在用AI之前，先确保你有访问以下工具的能力：

一、AI写周报（每天省25分钟）#

核心逻辑#

不是让AI替你编，是你把本周干的活粗略说出来，让AI帮你整理成漂亮的周报格式。

Claude/ChatGPT提示词模板#

你是一位专业的职场写作助手。请将以下工作内容整理成一份专业的周报/日报。

要求：
- 分点列出，重点突出
- 量化成果（有数字更好）
- 用正式但不过于刻板的语气
- 突出本周已完成的成果
- 下周计划单独列出

以下是我本周的工作内容（比较乱，帮我整理）：
{{把你做的事情随便写在这里，一段话也行、关键词也行}}

实测示例：

输入：

你这周做了：修复了3个线上bug，其中一个数据库死锁导致客户订单失败。开始了新项目"用户画像系统"的数据库设计。跟产品团队开了两次需求评审会。给新同事做了一次code review培训。

输出：

>> 本周工作完成情况

1. **线上Bug修复**：修复3个线上问题，含1个数据库死锁引起的客户订单失败（紧急处理，恢复时长<30分钟）
2. **用户画像系统**：启动新项目数据库设计，完成核心表结构初版
3. **需求评审**：参与产品团队2次需求评审会，明确8个技术要点
4. **团队支持**：完成1次新员工Code Review培训（参与5人）

>> 下周计划

1. 完成用户画像系统数据库设计评审
2. 开始设计用户画像系统的API接口
3. 持续跟踪线上系统稳定性

进阶用法：AI周报自动生成流#

用Claude的Projects或ChatGPT的GPTs功能，可以创建一个专门的周报生成器：

  1. 在ChatGPT中创建一个GPT，设定角色为"周报助手"
  2. 把你的日常工作内容（Jira工单、Git commits、邮件）丢进去
  3. AI自动按周报格式输出

这项功能需要ChatGPT Plus订阅（$20/月）。推荐搭配龙猫云或万达云使用，每月¥15-17搞定网络 + ChatGPT全功能访问。

二、AI做PPT（每天省1.5小时）#

方案一：Gamma.app（强烈推荐）#

Gamma.app 是目前最好的AI生成PPT工具，没有之一。

优势：

  • 一句主题描述 → 自动生成完整PPT（含大纲、配图、排版）
  • 支持中文内容
  • 免费版就能用（生成有限制）
  • 可导出为PPT/PDF格式

操作步骤：

  1. 打开 gamma.app，点击"Create with AI"
  2. 输入主题（中文就行），比如：“人工智能在电商行业的应用”
  3. 选择风格模板（Professional / Creative / Minimal）
  4. Gamma自动生成大纲 → 确认 → 生成完整PPT
  5. 手动微调细节文字和图片
  6. 导出为PPT文件

提示词模板：

帮我生成一份关于[你的主题]的演示文稿。
要求：
- 8-10页
- 受众：公司管理层
- 风格：专业简洁
- 语言：中文
- 每页不要太多字，每个点不超过20字

方案二：用ChatGPT/Claude写大纲 + 手动做#

如果不想用Gamma，也可以用AI写大纲你来做：

你是一位PPT内容策划专家。请为以下主题设计一份演示文稿的内容框架。

主题：[填写你的主题]
受众：[填写听众群体]
时长：[填写预计演讲时长]

要求：
1. 设定一个有记忆点的演讲主线
2. 每页只讲一个核心观点（不超过20字）
3. 每页都有具体的数据/案例支撑（编的也可以，标注"待替换"）
4. 最后一页是行动号召

请输出每页的标题和核心要点，以及建议的配图方向。

方案三（进阶）：Napkin AI + 截图流#

Napkin AI 可以将文字自动转成信息图。配上Gamma生成的PPT，效果翻倍。

工作流：

  1. Gamma生成PPT初稿 → 2. Napkin AI将关键数据转成图表 → 3. 截图插入PPT → 4. 导出完成

这是AI效率提升最明显的场景。实测下来，熟手用AI写代码效率提升至少3倍。

Claude Code：最强AI编程助手#

Claude Code是Anthropic推出的终端版AI编程工具，直接在你的项目里运行，能读文件、改代码、跑git。

常用指令示例：

# 写一个新功能
/start 帮我写一个用户登录的JWT验证中间件，Express + TypeScript

# 修bug
Debug一下为什么/api/login返回503，查服务端日志

# 重构
把这个controller拆分一下，职责太多了

# 写测试
给userService.ts写单元测试，覆盖率80%以上

工作流（已验证）：

  1. 把需求用自然语言告诉Claude Code
  2. Claude Code分析项目结构 → 写代码 → 自动测试
  3. 你只需要review结果，验收修改

需要Claude访问权限？搭配龙猫云或WgetCloud使用，Claude对话完全无障碍。

GitHub Copilot：日常编码搭档#

如果你在用VS Code或JetBrains，Copilot是最低门槛的AI编程助手：

常用场景：

  • 写重复性代码（CRUD、测试用例）：直接写注释，Copilot补全
  • 查API用法：不用翻文档，Copilot inline suggestion直接给你写
  • SQL查数据：写个注释"–查询本月订单TOP10"，Copilot帮你写SQL

AI编程的核心技巧#

最重要的一个原则：不要等AI替你做决定。

最佳用法：

  1. 你负责设计（架构/逻辑） → 你告诉AI要做什么
  2. AI负责执行（写代码） → AI生成实现代码
  3. 你负责校验（review/测试） → 你确认代码正确

提示词模板：

我用[技术栈]，要实现[具体功能]。
约束条件：
- 错误处理要完整
- 性能优先（O(n) > O(n²)）
- 不要引入新依赖
- 写TypeScript类型定义
- 包含单元测试

功能描述：[详细描述]

常见问题#

Q：这些AI工具国内都能用吗？#

A：ChatGPT、Claude、Gamma.app、Cursor都需要梯子（代理）才能访问。GitHub Copilot在国内可以直连，但速度较慢。建议配置一个稳定的代理服务——文中推荐的龙猫云、万达云都经过实测，一次配置所有AI工具全通。

Q：免费的AI够用吗？#

A：够用但有限制。ChatGPT免费版可以用GPT-3.5和有限次数的GPT-4o，写周报完全OK。Claude免费版有每日次数限制，重度使用建议付费（$20/月）。写代码建议用Claude Code（Pro订阅）+ Copilot（$10/月），月投入$30左右，换来每天省3小时。

Q：AI写的内容能直接用吗？#

A：不能照搬。 AI生成的周报、PPT、代码，都需要人工review一遍。AI擅长"搭框架"和"写草稿"，最后的把关和润色得自己来。尤其是周报——AI不知道你项目中那些微妙的细节，你需要在AI基础上补充真实信息。

Q：写代码用AI，代码质量靠得住吗？#

A：取决于你review得多认真。AI生成的代码一般结构上没问题，但可能有：

  • 过度工程（写了很多你根本不需要的抽象）
  • 安全漏洞（没做输入验证、SQL注入防护）
  • 依赖版本过旧

核心规则：信任但验证。 每个AI生成的commit都diff一下再提交。

写在最后#

AI效率工具的核心理念不是"AI替你工作"，而是**“AI帮你把重复劳动省掉，你专注于更有价值的事情”**。

  • 不想写周报？让AI帮你写框架，你往里填关键信息
  • 不想做PPT？让AI出初稿，你只做最后的美化
  • 不想写重复代码？让AI生成模板，你专注架构设计

用上AI和不用的区别，不是30%的效率差——是3倍的差距。
