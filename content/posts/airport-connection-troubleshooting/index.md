> 用机场最糟心的体验是什么？不是速度慢，而是**昨天还好好的，今天突然连不上了**。节点全红、延迟几千、订阅加载失败——这篇文章帮你从零开始排查，90%的问题不用找客服，自己就能搞定。

## 两步速查：先定位问题类型

遇到连不上，先别急着重装客户端。坐下来花30秒判断是什么问题：

| 现象 | 大概率原因 | 解决难度 |
|:----|:----------|:--------:|
| 节点全红（全部超时） | 订阅链接失效 / 运营商阻断 | ⭐ 简单 |
| 部分节点能用但慢 | 晚高峰 / 个别节点故障 | ⭐ 简单 |
| 能连上但打不开特定网站 | DNS污染 / 网站被墙升级 | ⭐⭐ 中等 |
| 客户端闪退 / 报错 | 客户端版本问题 | ⭐ 简单 |
| 所有节点时快时慢 | 本地网络问题 / 机场负载高 | ⭐⭐ 中等 |

## 第一步：检查客户端（最简单，先做）

### 1.1 订阅链接是否过期

机场订阅一般按月/季/年付费。很多用户的「连不上」只是**订阅过期了**。

**怎么查：**
- 打开机场官网 → 登录 → 查看剩余天数
- 如果显示「已过期」或「流量为0」→ 续费即可

⚠️ **注意：** 续费后通常需要**手动更新订阅**（在客户端点「更新订阅」按钮），不是续费完自动好的。

### 1.2 更新订阅链接

即使没过期，订阅链接也可能因服务器端升级而失效：

- **Clash Meta / Clash Verge**: 点击「更新订阅」或重启客户端自动拉取
- **Stash (iOS)**: 下拉刷新订阅
- **Surge**: 策略组 → 更新外部资源
- **Sing-Box**: 重新载入配置文件

**判断标准：** 更新后如果节点从「全红」变成「有延迟数值」→ 就是订阅过期或链接失效的问题，已解决 ✅

### 1.3 切换节点试试

有时候只是某个节点挂了，不是整个机场的问题：

- 别死磕「香港节点」
- 试试日本、新加坡、美国等不同地区的节点
- 试试「自动选择」或「故障转移」策略组

> 💡 **经验之谈：** 晚高峰（晚上8-11点）香港节点负载最高，这时候切到日本或新加坡往往速度更好。

## 第二步：检查网络环境

如果客户端没问题，检查你的本地网络。

### 2.1 先测本地网络

关掉代理，直接打开百度/微博：**能打开吗？**

| 本地网络 | 说明 |
|:---------|:-----|
| ✅ 百度正常打开 | 本地网络正常，问题在代理层面 |
| ❌ 百度也打不开 | 你家宽带有问题 → 重启路由器/光猫 |

### 2.2 重启大法（真的有用）

顺序很重要：

```
① 关闭机场客户端
② 重启路由器（拔电源等30秒再插上）
③ 等待路由器完全启动（指示灯正常）
④ 重启电脑/手机
⑤ 重新打开客户端 → 更新订阅 → 连接
```

**为什么有用？** 路由器长期运行会产生 DNS 缓存错误、NAT 表溢出等问题，重启能清空这些。很多「突然连不上」重启路由器就好了。

### 2.3 检查系统代理设置

有时候是系统代理配置混乱：

**Windows：**
- 设置 → 网络和互联网 → 代理
- 确保「使用代理服务器」为**关闭**状态（机场客户端自己管理代理，不需要系统代理开着）

**macOS：**
- 系统偏好设置 → 网络 → 高级 → 代理
- 确保所有代理选项都是**未勾选**状态

**手机：**
- 检查是否开启了「VPN」或「代理」开关（有些机场客户端会额外装一个VPN配置文件，和客户端本身冲突）

## 第三步：DNS 问题排查

DNS 是常见的「能连上但打不开」的罪魁祸首。

### 3.1 换 DNS

把电脑/手机的 DNS 改成公共 DNS：

| DNS | 地址 | 特点 |
|:----|:-----|:-----|
| 阿里 DNS | 223.5.5.5 / 223.6.6.6 | 国内快，稳定 |
| DNSPod | 119.29.29.29 | 腾讯系，延迟低 |
| Cloudflare | 1.1.1.1 / 1.0.0.1 | 全球通用，隐私好 |
| Google | 8.8.8.8 / 8.8.4.4 | 最稳定但国内可能慢 |

### 3.2 清 DNS 缓存

```bash
# Windows
ipconfig /flushdns

# macOS
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

# Linux
sudo systemd-resolve --flush-caches
# 或
sudo resolvectl flush-caches
```

搞定后重启客户端再试。

## 第四步：协议和端口问题

### 4.1 检查客户端代理模式

| 模式 | 适用场景 |
|:----|:--------|
| **Rule（规则）** | ✅ 默认推荐——国内直连、国外走代理 |
| **Global（全局）** | 所有流量走代理——适合访问被墙严重的网站 |
| **Direct（直连）** | 全部直连——排查用 |

**排查技巧：** 如果 Rule 模式下某个网站打不开，切成 Global 试试。Global 能打开 → 规则配置有问题（更新订阅可以修复）。

### 4.2 TUN 模式

如果你用的是 Clash 系客户端（Clash Verge / Clash Meta），开启 **TUN 模式** 能解决大部分「非浏览器软件连不上」的问题：

- TUN 模式把系统所有流量都接管（不只是浏览器）
- 适合：游戏、命令行工具、微信等不走系统代理的应用
- 开启方法：客户端设置 → 开启 TUN / 虚拟网卡

> ⚠️ TUN 模式和某些杀毒软件/防火墙冲突，如果开启后网络异常，关闭 TUN 试试。

## 第五步：进阶排查

### 5.1 检查运营商是否阻断

国内三大运营商（电信/联通/移动）对 VPN 流量的态度不同：

| 运营商 | 阻断程度 | 说明 |
|:-------|:--------|:-----|
| 联通 | 较轻 | 大部分机场能用 |
| 电信 | 中等 | 晚高峰可能 QoS（限速） |
| 移动 | 较重 | 国际出口窄，容易断 |

**自查：** 手机切 4G/5G 热点试试能否连接 → 如果能，说明宽带有问题；如果也不能，说明机场本身有问题。

### 5.2 换协议/端口

有些机场支持多种协议（Shadowsocks / V2Ray / Trojan）和端口：

- 在机场官网找到「高级设置」或「节点配置」
- 试试换端口（如 443 → 80 → 8080）
- 或者换协议类型

**适用场景：** 公司/学校网络限制了特定端口（只开 80/443），这时候普通端口连不上是正常的。

### 5.3 抓包看真实错误

如果以上都试过还不行，看客户端日志：

**Clash Verge：**
- 设置 → 日志 → 查看最新日志
- 搜索 `error` 或 `timeout`

**常见错误日志：**

| 日志 | 含义 | 解决 |
|:----|:-----|:-----|
| `connection refused` | 节点端口被拒 | 节点挂了，换节点 |
| `i/o timeout` | 连接超时 | 网络问题或节点挂了 |
| `TLS handshake failed` | TLS 握手失败 | 运营商阻断 / 节点被墙 |
| `proxy was not found` | 规则找不到节点 | 更新订阅 |

## 第六步：终极手段

### 6.1 重装客户端

有时候客户端配置文件损坏，重装能解决：

1. 完全卸载客户端（包括配置文件和缓存）
2. 重启电脑
3. 重新下载安装最新版本
4. 导入订阅重新配置

### 6.2 换机场

如果以上**全部试过**还连不上——可能是机场本身出了问题。判断标准：

- 同一机场多个节点在不同设备、不同网络下都连不上

## 常见问题 FAQ

**Q：所有节点都显示超时，但订阅没过期，怎么办？**
A：先重启路由器 → 在客户端点「更新订阅」→ 重试。还不行就切手机热点测试——手机能连说明你家宽带阻断了，需要机场换端口或协议。

**Q：Clash Verge 提示「订阅解析失败」？**
A：订阅链接可能失效了。登录机场官网重新生成订阅链接，在客户端粘贴替换旧的。

**Q：连接机场后国内网站打不开？**
A：检查是不是开了「全局模式」。切成「规则模式」——国内直连、国外走代理。如果已经是规则模式，更新订阅（规则文件可能过期了）。

**Q：为什么白天正常，晚上就卡？**
A：晚高峰（8-11点）所有用户都在用。这是正常的。建议：换到日本/新加坡节点（负载比香港低），或选择有 MPTCP 多路复用技术的机场（如自由猫）。

**Q：所有方法都试过了还是连不上？**
A：先确认不是机场本身的问题——去机场官网看公告或问客服。如果官网也打不开 → 换一家机场（参考上面的推荐表）。
