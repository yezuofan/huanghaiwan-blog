## 场景一：周报撰写（最适合AI的场景）

周报是最高频的AI提效场景，反馈周期短、效果立竿见影。

### 传统方式 vs AI方式

| 对比 | 传统方式 | AI方式 |
|:----|:--------|:--------|
| 耗时 | 30-60分钟 | 5-10分钟 |
| 质量 | 取决于当天状态 | 稳定输出 |
| 痛苦程度 | 高（挤牙膏式回忆） | 低（聊天式输出） |

### 三步提效工作流

**第1步：记录（上班时随手记）**

用一个文档（Notion/飞书/甚至微信文件传输助手），随手记下今天做了什么：

```
- 完成了用户调研报告（初稿）
- 修复了登录页面的bug
- 参加了产品评审会，提出了3个优化建议
```

不需要格式，不需要完整，就记关键词。这是AI的原料。

**第2步：整理（1分钟）**

把今天的记录整理成一个结构化的清单：

```
1. 完成用户调研报告初稿，输出给产品
2. 修复登录页面包涵登录失效问题
3. 产品评审会：提出导航优化、数据看板、通知中心3个需求
```

**第3步：AI生成（3分钟）**

把清单发给AI，让它帮你写周报：

```
请帮我把以下工作记录整理成周报格式，语言要简洁专业，突出成果和价值：

[粘贴你的工作记录]

要求：
1. 每条内容后注明对业务的价值
2. 用动词开头（完成了/优化了/推进了）
3. 不超过300字
4. 语气积极但不夸张
```

### AI周报提示词模板（直接复制用）

```
请帮我把以下工作记录整理成周报格式：

[粘贴工作内容]

要求：
1. 每条用「动词+成果+数据」的格式（如：完成了XX报告，输出给XX部门使用）
2. 成果类内容用数字量化（如：覆盖100+用户、响应时间降低40%）
3. 不超过300字
4. 语气专业，适合给主管看
```

## 场景二：PPT制作（AI帮你做80%的苦力活）

做PPT最耗时的不是设计，是内容梳理和排版。AI能接管大部分工作。

### AI辅助PPT制作流程

```
原始素材 → AI整理框架 → AI生成PPT（Gamma/Beautiful.ai） → 人工微调
```

### 具体步骤

**第1步：准备素材（5分钟）**

把你有的材料都丢到一个文档里：
- 会议记录
- 数据报告
- 竞品分析
- 领导的要求

**第2步：让AI帮你梳理框架**

```
请帮我把以下材料整理成一个PPT框架，要求：
1. 控制在10页以内
2. 每页有明确的标题和3个以内的要点
3. 开头有背景/目标，结尾有结论/建议
4. 输出结构用Markdown表示

[粘贴材料]
```

**第3步：用Gamma自动生成PPT**

[Gamma](https://gamma.app/) 是目前最智能的AI PPT工具：

1. 把AI输出的Markdown框架粘贴到Gamma
2. 选择主题风格（可选：商务/创意/简约）
3. 自动生成10-15页PPT
4. 下载或在线分享

**第4步：微调（10-15分钟）**

AI生成的PPT通常：
- ✅ 内容结构清晰
- ✅ 数据可视化好看
- ❌ 细节数据可能不准
- ❌ 配色需要按公司标准调整

重点检查：数字、人名、结论。

### 替代工具推荐

| 工具 | 优点 | 适合场景 |
|:----|:----|:---------|
| [Gamma](https://gamma.app/) | AI生成能力强，设计好看 | 快速原型/不需要品牌规范 |
| [Beautiful.ai](https://www.beautiful.ai/) | 模板多，自动化对齐 | 企业内用/有品牌规范 |
| [Canva AI](https://www.canva.com/) | 简单易用，中文支持好 | 非设计师/临时需要 |

## 场景三：编程辅助（AI让代码质量提升一个档次）

编程是AI提效最明显的场景，但很多人用错了方式。

### 推荐的AI编程工作流

| 任务类型 | 工具选择 | 使用技巧 |
|:--------|:--------|:---------|
| 写新功能 | Claude Code / Windsurf | 直接描述需求，让AI写代码 |
| Debug | DeepSeek R1 | 把报错信息贴进去，让它分析根因 |
| Code Review | Claude | 让它检查性能/安全/可维护性问题 |
| 写测试 | GPT-4o / Claude | 指定测试框架和覆盖率要求 |
| 重构老代码 | Claude | 先让它理解代码，再提重构建议 |

### 编程提示词模板

**写新功能：**
```
请用[语言]写一个[功能描述]，要求：
1. 使用[框架/库]
2. 输入格式：[描述]
3. 输出格式：[描述]
4. 包含错误处理
5. 代码要加注释

[具体需求描述]
```

**Debug（最常用）：**
```
我的代码运行报错，错误信息如下：

[粘贴完整报错信息]

我的代码：
[粘贴相关代码]

请帮我：
1. 分析报错根本原因
2. 给出修复后的完整代码
3. 解释修复逻辑
```

**Code Review：**
```
请审查以下[语言]代码，重点关注：
1. 性能瓶颈
2. 潜在bug
3. 安全漏洞（如SQL注入、XSS等）
4. 代码可维护性
5. 命名规范

[粘贴代码]
```

### 选Claude还是DeepSeek？

| 场景 | 推荐 | 原因 |
|:----|:----|:-----|
| 写复杂业务逻辑 | **Claude** | 上下文理解力强，能理解业务背景 |
| 数学/算法/数据结构 | **DeepSeek R1** | 推理能力强，数学逻辑更严谨 |
| 简单脚本/工具 | **DeepSeek V3** | 速度快，免费足够 |
| 代码审查 | **Claude** | 表达更清晰，建议更有条理 |

## 场景四：内容创作（AI不是替代，是放大）

内容创作是AI介入最深的领域，但也是最容易出问题的地方。

### 正确用法 vs 错误用法

| 错误用法 | 正确用法 |
|:--------|:---------|
| 让AI直接写完整文章 | 让AI帮你：列大纲/优化标题/润色文字 |
| 100% AI生成不修改 | AI出初稿，人工调整节奏和情感 |
| 什么内容都用AI | 深度观点类内容需要人写，AI做辅助 |
| 忽视事实核查 | AI会编造数据，必须人工核实 |

### 内容创作工作流

**步骤1：Perplexity做信息收集（5分钟）**

想写一个话题，先让Perplexity帮你整理这个话题的核心信息：

```
搜索：关于[主题]的核心信息，包括：
1. 最新发展动态
2. 主流观点和争议
3. 常用数据和案例
4. 普通人最关心的问题

输出格式：结构化的信息摘要
```

**步骤2：Claude做深度分析和框架（10分钟）**

```
请帮我分析以下材料，输出：
1. 3个最有价值的切入角度（选择一个你最有话说的）
2. 文章大纲（6-8个段落）
3. 每个段落的关键词和数据支撑点
4. 一个吸引人的标题备选

[粘贴材料]
```

**步骤3：AI写初稿（10-15分钟）**

```
请帮我写一篇关于[主题]的文章，要求：
1. 目标读者：[描述]
2. 字数：800-1200字
3. 风格：[专业/通俗/幽默...]
4. 包含至少3个具体案例或数据
5. 开头要有钩子，结尾要有行动指引

文章大纲：
[粘贴大纲]
```

**步骤4：人工润色和发布（10分钟）**

- 调整文章节奏和语气
- 加入个人经历或独特观点（AI写不出来的东西）
- 检查事实和数据
- 加入合适配图
- 发布到对应平台

### AI配套工具推荐

| 工具 | 用途 |
|:----|:-----|
| [Perplexity](https://www.perplexity.ai/) | 信息收集和背景研究 |
| [Claude](https://claude.ai/) | 文章框架和深度分析 |

## 场景五：数据分析（AI让Excel小白也能做分析）

数据分析是AI提效潜力最大的场景，但也是最需要谨慎的地方。

### 数据分析工作流

**第1步：让AI帮你理解数据**

```
我有一份[Excel/CSV]数据，文件路径：[描述]
请帮我：
1. 描述数据整体情况（行数、列数、每个字段的含义）
2. 找出明显的数据质量问题（缺失值、异常值、格式错误）
3. 提出3个最有价值的分析方向

[简要描述业务背景]
```

**第2步：让AI生成分析代码**

```
请用Python（pandas + matplotlib）帮我：
1. 读取并清洗这份数据
2. 做[描述你的分析需求，如：按月统计销售额趋势]
3. 生成可视化图表

数据文件路径：[描述]
```

**第3步：AI解读结果**

```
请帮我解读以下分析结果：
[粘贴分析输出或截图]

问题：
1. 这个数据揭示了什么关键信息？
2. 有哪些反常或需要关注的点？
3. 基于这些数据，你建议下一步做什么？
```

### 数据分析的坑（AI也会出错）

| 坑 | 说明 | 避免方法 |
|:--|:----|:---------|
| 数据幻觉 | AI可能编造不存在的数字 | 所有数字都要核对原始数据 |
| 统计方法错误 | AI可能用错误的统计方法 | 复杂的统计咨询专业人士 |
| 忽略业务逻辑 | 数据上相关≠业务上相关 | 结合业务背景判断 |
| 过度依赖AI | AI只是工具，决策权在你自己 | AI是辅助，不是决策者 |

## 常见问题（FAQ）

### Q：AI会不会让我变懒？

**会的，如果你只用AI聊天而不建立工作流。**

AI提效的关键是**建立固定的流程**，而不是每次都临时问。固定模板、固定流程、固定工具链，这样才能把节省的时间真正转化为效率。

### Q：我应该从哪个场景开始？

**从周报开始，这是反馈周期最短、效果最明显的场景。**

流程：
1. 今天上班随手记录工作内容（30秒/天）
2. 周末让AI帮你整理成周报（5分钟/周）
3. 节省的时间用来做复盘和规划

### Q：需要付费吗？

| 工具 | 免费额度 | 付费 |
|:----|:--------|:-----|
| DeepSeek | 完全免费 | ❌ 不需要 |
| Perplexity | 有限次 | $20/月 |
| Claude | 有限次 | $20/月 |
| Gamma | 有限次 | $12/月 |
| Suno | 有限次 | $10/月 |

**建议先免费用，感受到价值再付费。** DeepSeek完全免费，日常任务够用。

### Q：AI生成的内容可以直接用吗？

**不能直接用，必须人工审核。**

AI的三大问题：
1. **事实错误**——AI会编造数据，必须核实
2. **语气问题**——AI写的文字缺乏个人风格
3. **深度不足**——AI能写框架，写不出独特观点

正确姿势：AI出初稿，人工做三件事——核实事实、调整语气、补充观点。
